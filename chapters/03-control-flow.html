Chapter 3: Control Flow

Making Decisions with if/else

Control flow statements let your program make decisions and execute different code based on conditions. The most fundamental decision-making statement is if.

Basic if Statement

#include <stdio.h>

int main() {
    int age = 18;
    
    if (age >= 18) {
        printf("You are eligible to vote.\n");
    }
    
    return 0;
}

if-else Statement

#include <stdio.h>

int main() {
    int marks = 85;
    
    if (marks >= 60) {
        printf("You passed!\n");
    } else {
        printf("You failed. Better luck next time.\n");
    }
    
    return 0;
}
üí° Key Concepts
  • Condition must be in parentheses: if (condition)
  • Use curly braces {} even for single statements (good practice)
  • The else part is optional

else-if Ladder

When you have multiple conditions to check, use the else-if ladder:

#include <stdio.h>

int main() {
    int marks;
    printf("Enter your marks: ");
    scanf("%d", &marks);
    
    if (marks >= 90) {
        printf("Grade: A+\n");
    } else if (marks >= 80) {
        printf("Grade: A\n");
    } else if (marks >= 70) {
        printf("Grade: B\n");
    } else if (marks >= 60) {
        printf("Grade: C\n");
    } else if (marks >= 40) {
        printf("Grade: D\n");
    } else {
        printf("Grade: F (Fail)\n");
    }
    
    return 0;
}
üéØ How it Works

The conditions are checked in order. As soon as one condition is true, that block executes and the rest are skipped.

Comparison Operators

These operators compare two values and return true (1) or false (0):

Operator Meaning Example Result (if a=5, b=3)
== Equal to a == b false (0)
!= Not equal to a != b true (1)
> Greater than a > b true (1)
< Less than a < b false (0)
>= Greater than or equal a >= b true (1)
<= Less than or equal a <= b false (0)
‚ö†Ô∏è Common Pitfall: = vs ==

NEVER confuse assignment (=) with comparison (==)!

  • if (x = 5) ‚ùå - This assigns 5 to x, then checks if 5 is true (it is)
  • if (x == 5) ‚úÖ - This compares x with 5

Logical Operators

Combine multiple conditions using logical operators:

Operator Meaning Example True when
&& AND a && b Both a AND b are true
|| OR a || b Either a OR b (or both) is true
! NOT !a a is false

Practical Examples

#include <stdio.h>

int main() {
    int age = 20, income = 30000;
    
    // AND operator
    if (age >= 18 && income >= 25000) {
        printf("Eligible for credit card\n");
    }
    
    // OR operator
    if (age < 18 || age > 65) {
        printf("Special discount available\n");
    }
    
    // NOT operator
    if (!(age < 18)) {  // Same as: age >= 18
        printf("Adult\n");
    }
    
    return 0;
}
üí° Short-circuit Evaluation
  • AND (&&): If first condition is false, second isn't evaluated
  • OR (||): If first condition is true, second isn't evaluated

switch Statement

When comparing a single variable against multiple exact values, switch is cleaner than multiple if-else statements:

#include <stdio.h>

int main() {
    int day;
    printf("Enter day number (1-7): ");
    scanf("%d", &day);
    
    switch (day) {
        case 1:
            printf("Monday\n");
            break;
        case 2:
            printf("Tuesday\n");
            break;
        case 3:
            printf("Wednesday\n");
            break;
        case 4:
            printf("Thursday\n");
            break;
        case 5:
            printf("Friday\n");
            break;
        case 6:
            printf("Saturday\n");
            break;
        case 7:
            printf("Sunday\n");
            break;
        default:
            printf("Invalid day number!\n");
    }
    
    return 0;
}

Menu-driven Program Example

#include <stdio.h>

int main() {
    int choice;
    float num1, num2;
    
    printf("Calculator Menu:\n");
    printf("1. Addition\n");
    printf("2. Subtraction\n");
    printf("3. Multiplication\n");
    printf("4. Division\n");
    printf("Enter your choice: ");
    scanf("%d", &choice);
    
    printf("Enter two numbers: ");
    scanf("%f %f", &num1, &num2);
    
    switch (choice) {
        case 1:
            printf("Result: %.2f\n", num1 + num2);
            break;
        case 2:
            printf("Result: %.2f\n", num1 - num2);
            break;
        case 3:
            printf("Result: %.2f\n", num1 * num2);
            break;
        case 4:
            if (num2 != 0) {
                printf("Result: %.2f\n", num1 / num2);
            } else {
                printf("Error: Division by zero!\n");
            }
            break;
        default:
            printf("Invalid choice!\n");
    }
    
    return 0;
}
‚ö†Ô∏è Don't Forget break!

Without break, execution "falls through" to the next case. This is usually not what you want!

When to Use switch vs if-else

  • Use switch when: Comparing one variable against multiple exact values
  • Use if-else when: Complex conditions, ranges, or different variables

Nested Conditions

You can place if statements inside other if statements:

#include <stdio.h>

int main() {
    int age, license;
    
    printf("Enter age: ");
    scanf("%d", &age);
    
    if (age >= 18) {
        printf("Do you have a driving license? (1=Yes, 0=No): ");
        scanf("%d", &license);
        
        if (license == 1) {
            printf("You can drive!\n");
        } else {
            printf("You need to get a driving license first.\n");
        }
    } else {
        printf("You are too young to drive.\n");
    }
    
    return 0;
}
üí° Keep It Simple

Avoid too many nested levels (more than 3-4). Use logical operators or separate functions instead.

Common Errors with Control Flow

‚ùå Assignment instead of comparison

Error: if (x = 5)
Fix: if (x == 5)
Why: Single = assigns value; double == compares values.

‚ùå Missing break in switch

Error: Cases fall through to next case
Fix: Add break; after each case
Why: Without break, execution continues to next case.

‚ùå Semicolon after if condition

Error: if (x > 5); printf("Big");
Fix: if (x > 5) printf("Big");
Why: Semicolon creates an empty statement, breaking the if logic.

‚ùå Floating point equality

Error: if (price == 19.99)
Fix: if (fabs(price - 19.99) < 0.01)
Why: Floating point numbers aren't stored exactly.
