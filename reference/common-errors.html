Common C Programming Errors & Solutions

A comprehensive guide to identifying, understanding, and fixing common errors in C programming.

Types of Errors

Error Type When it Occurs Detection Example
Syntax Error During compilation Compiler catches it Missing semicolon
Runtime Error During execution Program crashes Division by zero
Logic Error During execution Wrong output Incorrect formula
Warning During compilation Compiler warns Unused variable

Compilation (Syntax) Errors

1. Missing Semicolon

‚ùå Error
int x = 5  // Missing semicolon
printf("%d", x);

Fix: Add semicolon after each statement

int x = 5;  // Fixed
printf("%d", x);

2. Missing Header Files

‚ùå Error
// Missing #include <stdio.h>
int main() {
    printf("Hello");  // Error: 'printf' undeclared
    return 0;
}

Fix: Include necessary headers

#include <stdio.h>  // Fixed
int main() {
    printf("Hello");
    return 0;
}

3. Mismatched Parentheses/Braces

‚ùå Error
if (x > 5 {  // Missing closing parenthesis
    printf("Greater");
}  // Missing opening brace

Fix: Match all brackets properly

if (x > 5) {  // Fixed
    printf("Greater");
}

4. Undeclared Variables

‚ùå Error
int main() {
    age = 20;  // Error: 'age' undeclared
    printf("%d", age);
    return 0;
}

Fix: Declare variables before use

int main() {
    int age;   // Fixed: declare first
    age = 20;
    printf("%d", age);
    return 0;
}

Runtime Errors

1. Division by Zero

‚ùå Error
int a = 10, b = 0;
int result = a / b;  // Runtime error: division by zero

Fix: Check for zero before division

int a = 10, b = 0;
if (b != 0) {
    int result = a / b;
    printf("Result: %d", result);
} else {
    printf("Error: Division by zero!");
}

2. Array Index Out of Bounds

‚ùå Error
int arr[5] = {1, 2, 3, 4, 5};
printf("%d", arr[5]);  // Error: index 5 doesn't exist (0-4 only)

Fix: Use valid array indices

int arr[5] = {1, 2, 3, 4, 5};
printf("%d", arr[4]);  // Fixed: use index 0-4

3. Infinite Loops

‚ùå Error
int i = 0;
while (i < 10) {
    printf("%d ", i);
    // Missing i++, causes infinite loop
}

Fix: Update loop variable

int i = 0;
while (i < 10) {
    printf("%d ", i);
    i++;  // Fixed: increment i
}

Logic Errors

1. Assignment vs. Comparison

‚ùå Error
int x = 5;
if (x = 10) {  // Using = instead of ==
    printf("x is 10");
}

Fix: Use == for comparison

int x = 5;
if (x == 10) {  // Fixed: comparison operator
    printf("x is 10");
}

2. Wrong Loop Conditions

‚ùå Error
// Want to print 1 to 10
for (int i = 1; i < 10; i++) {  // Only prints 1-9
    printf("%d ", i);
}

Fix: Use correct condition

for (int i = 1; i <= 10; i++) {  // Fixed: prints 1-10
    printf("%d ", i);
}

3. Off-by-One Errors

‚ùå Error
int arr[5] = {1, 2, 3, 4, 5};
// Want to print all elements
for (int i = 0; i < 4; i++) {  // Misses last element
    printf("%d ", arr[i]);
}

Fix: Use correct bounds

int arr[5] = {1, 2, 3, 4, 5};
for (int i = 0; i < 5; i++) {  // Fixed: all elements
    printf("%d ", arr[i]);
}

Input/Output Errors

1. Missing & in scanf

‚ùå Error
int age;
scanf("%d", age);  // Missing &

Fix: Use address operator

int age;
scanf("%d", &age);  // Fixed

2. Wrong Format Specifiers

‚ùå Error
float price = 19.99;
printf("Price: %d", price);  // Using %d for float

Fix: Use correct specifier

float price = 19.99;
printf("Price: %.2f", price);  // Fixed: %f for float

3. Character Input After Numeric Input

‚ùå Error
int num;
char ch;
scanf("%d", &num);
scanf("%c", &ch);  // Reads leftover newline

Fix: Add space before %c

int num;
char ch;
scanf("%d", &num);
scanf(" %c", &ch);  // Fixed: space consumes whitespace

Memory-Related Errors

1. Uninitialized Variables

‚ùå Error
int sum;  // Contains garbage value
printf("Sum: %d", sum);  // Prints random number

Fix: Initialize variables

int sum = 0;  // Fixed: initialized
printf("Sum: %d", sum);

2. Stack Overflow (Deep Recursion)

‚ùå Error
void recurse(int n) {
    printf("%d ", n);
    recurse(n + 1);  // No termination condition
}

Fix: Add base case

void recurse(int n) {
    if (n > 100) return;  // Fixed: base case
    printf("%d ", n);
    recurse(n + 1);
}

Debugging Strategies

1. Read Error Messages Carefully

  • Error messages usually indicate the line number
  • Look for keywords like "undeclared", "expected", "missing"
  • The actual error might be on the line before the reported line

2. Use printf() for Debugging

int x = 5, y = 10;
printf("Debug: x = %d, y = %d\n", x, y);  // Check values
int result = x + y;
printf("Debug: result = %d\n", result);

3. Comment Out Code Sections

Isolate problems by commenting out portions of code:

// Temporarily comment out problematic code
// problematic_function();
simple_function();

4. Check Boundary Conditions

  • Test with edge cases (0, negative numbers, maximum values)
  • Verify array bounds
  • Check loop termination conditions

5. Use a Debugger

Modern IDEs provide debuggers that allow you to:

  • Set breakpoints
  • Step through code line by line
  • Inspect variable values
  • Trace program execution

Error Prevention Tips

üí° Best Practices
  • Initialize variables: Always give variables initial values
  • Use meaningful names: studentAge vs sa
  • Check input validity: Validate user input before processing
  • Add comments: Explain complex logic
  • Test frequently: Compile and test after small changes
  • Use consistent formatting: Proper indentation helps spot errors
  • Break down complex problems: Write smaller, testable functions

Quick Error Reference

"expected ';'" Missing semicolon Add ; at end of statement
"undeclared" Variable/function not declared Declare variable or include header
"expected ')'" Mismatched parentheses Check all ( have matching )
"incompatible types" Type mismatch Check variable types and casting
"segmentation fault" Invalid memory access Check array bounds and pointers
